# MSR-HINE-2D

**Multiscale Recurrent Hierarchical Neural Emulator for stabilizing long-horizon autoregressive rollouts of chaotic PDEs — 2D forced Kolmogorov flow.**

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)

---

## Overview

Neural surrogates for chaotic PDE systems are accurate over a few steps but degrade rapidly during long autoregressive (self-fed) rollouts — they drift, blow up, or lose their energy spectrum. MSR-HINE attacks this by giving a neural operator a **multiscale recurrent memory**: slow, coarse-scale latent states that persist across the rollout and anchor the fast, fine-scale prediction, analogous to how a multirate numerical integrator treats slow and fast dynamics on different clocks.

This repository implements the **2D extension** of MSR-HINE, benchmarked on **forced Kolmogorov flow** — a canonical 2D turbulence system whose dual energy/enstrophy cascade makes it a natural stress test for a method built around scale separation.
---

## Method

| Component | Description |
|---|---|
| **State** | Vorticity field `ω ∈ ℝ^{256×256}`, periodic `[0, 2π]²` |
| **Backbone** | Hierarchical 2D U-Net (residual blocks, GroupNorm, SiLU, bottleneck self-attention) |
| **Spectral hierarchy** | Three nested radial low-pass bands: fine `\|k\| ≤ 85` (full field), medium `\|k\| ≤ 16` (stride 2), coarse `\|k\| ≤ 8` (stride 4) |
| **Recurrence** | Vector GRU on compact latents at medium and coarse scales; each evolves on its own clock |
| **Injection** | Decoded priors injected at matching U-Net stages via FiLM modulation + spatial concatenation |
| **Consistency** | `L_cons` (training only) — no inference-time posterior fusion |


---

## Testbed: 2D Forced Kolmogorov Flow

```
∂ₜω + (u·∇)ω = ν∇²ω + f,     f = −k_f cos(k_f y) − μω
```

| Parameter | Value |
|---|---|
| Reynolds number `Re` | 4000 |
| Forcing wavenumber `k_f` | 4 |
| Linear drag `μ` | 0.1 |
| Grid | 256 × 256 (2/3 dealiasing, max mode ≈ 85) |
| Snapshot spacing | ≈ τ_λ / 15  (Lyapunov time τ_λ ≈ 84 steps) |
| Training trajectories | 50 × 500 steps |
| Test trajectories | 10 × 1500 steps |

Data is generated by a self-contained pseudo-spectral solver (ETDRK4 time-stepping with an integrating factor for the viscous term); no external CFD code is required.

## Models and Ablations

The recurrence ablation is backbone-controlled on the U-Net. FNO is the explicit one-step baseline kept consistent with the 1D (KS, L96) experiments.

| Config | Backbone | Description |
|---|---|---|
| `fno_1step` | FNO | Explicit autoregressive reference: `ω_{t+1} = F(ω_t)` |
| `hine` | U-Net | No recurrence: staggered-horizon latent futures regenerated each step |
| `msr_hine` | U-Net | **Full model**: multi-rate recurrent hierarchy |
| `single_scale` | U-Net | One recurrent scale only |
| `no_multirate` | U-Net | All update strides = 1 |
| `no_topdown` | U-Net | Coarse→medium top-down conditioning removed |
| `no_consistency` | U-Net | Consistency loss `L_cons` removed |
| `no_contraction` | U-Net | Spectral-norm contraction safeguard off |
| `no_warmup` | U-Net | Recurrent states zero-initialized (no warmup window) |

## Results

Valid prediction horizon (VPH) in Lyapunov-time units τ_λ, measured after a 12-step warmup:

| Model | VPH (τ_λ) |
|---|---|
| HINE-L2 | 1.24 |
| **MSR-HINE** | **1.86** |

MSR-HINE also maintains tighter long-run energy spectra and stable enstrophy under free rollout beyond 10 τ_λ.

## Installation

```bash
git clone https://github.com/dmrigank/Multi-rate_recurrent_recurrence_hine.git
cd Multi-rate_recurrent_recurrence_hine
conda env create -f environment.yml
conda activate msr_hine
pip install -e .
```

Requires Python ≥ 3.10, PyTorch ≥ 2.1 with CUDA. A GPU with ≥ 16 GB VRAM is recommended for TBPTT training at 256² (gradient checkpointing keeps peak memory tractable).

## Usage

### 1. Generate data

```bash
python -m msr_hine.data.generate
```

Writes `data/kolmogorov/{train,val,test}.h5` (~10 GB total). Parameters are set in `configs/data/data_kolmogorov.yaml`.

### 2. Estimate Lyapunov time

```bash
python scripts/estimate_lyapunov.py
```

Writes `data/kolmogorov/lyapunov_estimate.json`. All VPH values are normalized by τ_λ.

### 3. Train

```bash
# Full MSR-HINE model
python -m msr_hine.train --config-name config model=model_msr_hine

# HINE baseline (no recurrence)
python -m msr_hine.train --config-name config model=model_unet_1step

# FNO baseline
python -m msr_hine.train --config-name config model=model_fno

# Ablation example
python -m msr_hine.train --config-name config model=model_msr_hine +ablation=no_topdown
```

### 4. Evaluate

```bash
python scripts/aggregate_results.py --run-dir outputs/<run_name>
```

Writes `metrics.json` with RMSE, ACC, VPH, and spectral error aggregated over all test trajectories.

### 5. Generate paper figures and tables

```bash
python scripts/plot_results.py --out-dir outputs/paper_figures_v5
python scripts/generate_tables.py --out-dir outputs/paper_figures_v5
```

### 6. Run full ablation sweep

```bash
bash scripts/run_ablations.sh
```

## Key Design Invariants

These encode the structural corrections over earlier HINE formulations:

1. **No inference-time posterior fusion.** `L_cons` is a training loss only. Re-encoding the model's own prediction at inference is circular and reinforces drift.
2. **No ground-truth signal on any inference-path input.** Ground truth enters only as loss targets.
3. **Stride-respecting latent losses.** `L_prior` and `L_cons` for level `l` are computed only on update steps (`n mod s_l == 0`).
4. **Multi-rate hold is real.** Off-stride levels do not evolve their GRU and their prior is held constant.
5. **Spectral truncation is fixed and nested.** `P^l` are radial Fourier low-pass projectors, not learned downsampling.
6. **Shared backbone for the recurrence ablation.** `hine` and `msr_hine` use the same U-Net; FNO is used only for `fno_1step`.
